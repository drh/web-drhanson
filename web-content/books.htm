Here, in no particular order, are my favorite books and software. Most of the books are computer science books, but a few are about writing, data graphics, flight, and polar exploration. Over the years, I've come back to these books many times, and I own two or three copies of some of them—one for home and one for the office. The software is what I install immediately when I get a new computer. The most important attribute of these software packages is that they just work.

Best Books

Writing Solid Code
Solid advice for C programmers on how to bullet-proof their code.
Art of Computer Programming, Vol. 1-3
Knuth's classic set of algorithms books. If it's not in here, it doesn't exist. The only downside is use of MIX assembly language as the implementation language, but rewriting the algorithms in higher-level languages forces one to understand the MIX code, which helps understand the algorithms better.
The Practice of Programming
Kernighan's and Pike's latest "how to program by example". Almost every example radiates elegance in both design and implementation. Every programmer should read this book.
Programming Pearls, 2/e
Jon Bentley's timeless advice on choosing algorithms, programming, and performance tuning. Basically, don't tune—at least until you actually know where your program is spending its time.
Software Tools
This 1976 classic is a bit dated now, but it remains one of the best examples of the Unix approach to program design.
The Elements of Style
If you own only one book on writing, it should be this one. Much writing, including nearly every computer science paper I've read, prattles on endlessly. Despite what you learned in 6th grade, more is not better, it's just more. Follow Strunk's and White's advice and you'll write better papers.
The Visual Display of Quantitative Data
Visual Explanations
Envisioning Information
Beautiful Evidence (to appear)
